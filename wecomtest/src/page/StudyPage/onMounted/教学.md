`onMounted` 是 **Vue 3 组合式 API（Composition API）中的生命周期钩子**，用于在组件**挂载完成后执行代码**。

---

## 一句话理解

👉 当组件的 DOM **已经渲染到页面上** 时，`onMounted` 就会执行

---

## 基本用法

```js
import { onMounted } from 'vue'

onMounted(() => {
  console.log('组件已经挂载完成')
})
```

---

## 它到底“什么时候”执行？

Vue 组件的生命周期大致是：

1. 创建组件（setup 执行）
2. 渲染虚拟 DOM
3. 挂载到真实 DOM
4. ✅ 执行 `onMounted`

---

## 常见使用场景

### 1️⃣ 请求后端数据（你现在做 Android + SpringBoot，很类似）

```js
onMounted(() => {
  fetch('/api/data')
    .then(res => res.json())
    .then(data => {
      console.log(data)
    })
})
```

👉 相当于：页面一加载就请求数据

---

### 2️⃣ 操作 DOM

```js
import { ref, onMounted } from 'vue'

const el = ref(null)

onMounted(() => {
  console.log(el.value) // 已经可以拿到真实 DOM
})
```

```html
<div ref="el"></div>
```

---

### 3️⃣ 初始化第三方库

比如：

* 图表（ECharts）
* 地图
* 动画库

---

## 和 Vue2 的对比

如果你学过 Vue2：

| Vue2      | Vue3        |
| --------- | ----------- |
| mounted() | onMounted() |

```js
// Vue2
mounted() {
  console.log('mounted')
}
```

```js
// Vue3
onMounted(() => {
  console.log('mounted')
})
```

---

## ⚠️ 注意点（很重要）

### ❌ 不要在 onMounted 之前操作 DOM

```js
// 错误 ❌
console.log(el.value) // 还没挂载，可能是 null
```

---

### ❗ setup 中就能写 onMounted

```js
export default {
  setup() {
    onMounted(() => {
      console.log('ok')
    })
  }
}
```

---

## 更深入一点（帮你建立理解）

你可以把它理解成：

👉 `onMounted = “页面已经加载好了，可以开始干活了”`

就像你写 Android：

* `onCreate()` → 初始化
* `onResume()` → 页面显示

👉 `onMounted` 更像是 “界面已经显示出来了”

---

## 总结

👉 `onMounted` = 组件挂载完成后执行的函数
👉 常用于：请求数据、操作 DOM、初始化功能
👉 是 Vue3 生命周期中最常用的钩子之一
