# Vue3 中 props 和 emit 的使用详解（父子组件通信）

在 Vue3 中，组件之间最常见的通信方式就是：

* **父组件 → 子组件：props**
* **子组件 → 父组件：emit**

本文通过一个简单例子，带你彻底理解它们的用法和本质。

---

# 一、props：父传子

首先，我们在子组件 `Child` 中定义 `props`，用于接收父组件传递的数据：

```js
const props = defineProps({
  msg: String
})
```

---

然后在父组件 `Parent` 中使用子组件：

```vue
<Child :msg="parentMsg" />
```

这里：

* `parentMsg` 是父组件的数据
* 通过 `:msg` 传递给子组件
* 子组件通过 `props.msg` 接收

👉 这就是 **props 的基本用法（单向数据流）**

---

# 二、问题：子组件如何修改父组件的数据？

一个很常见的需求：

> 比如：我双击子组件（像文件夹重命名），希望修改父组件中的数据。

---

## ❌ 错误思路（很多人会这样想）

* 在父组件放按钮去改值 ❌
* 或者试图在子组件直接修改 props ❌（Vue 不允许）

👉 因为：

> **props 是只读的，子组件不能直接修改父组件的数据**

---

# 三、解决方案：使用 emit（子传父）

Vue 提供了 `emit`，用于子组件向父组件“发送事件”。

---

## 1️⃣ 子组件：定义并触发事件

```js
const emit = defineEmits(['change'])

function changeMsg() {
  emit('change', '这是子组件发回去的新值')
}
```

这里做了两件事：

1. 定义事件：`change`
2. 触发事件，并携带数据：`'这是子组件发回去的新值'`

---

## 2️⃣ 父组件：监听事件

```vue
<Child :msg="parentMsg" @change="handleChange" />
```

---

## 3️⃣ 父组件：接收数据并更新

```js
function handleChange(newValue) {
  parentMsg = newValue
}
```

---

# 四、关键点（你原本容易出错的地方）

你原本的写法是：

```vue
@change="parentMsg = '这是子组件发回去的新值'"
```

这个写法的问题是：

* ❌ 没有接收子组件传递的数据
* ❌ 而是写死了一个固定值
* ❌ emit 传什么都没用

---

## ✅ 正确写法（核心）

```vue
@change="parentMsg = $event"
```

👉 `$event` 就是子组件 `emit` 传过来的值

---

# 五、更简洁的写法

你也可以不写函数，直接这样：

```vue
<Child :msg="parentMsg" @change="parentMsg = $event" />
```

---

# 六、本质理解（非常重要）

你可以把 Vue 的通信理解成函数调用：

```js
Child({
  msg: parentMsg,
  onChange: (value) => {
    parentMsg = value
  }
})
```

对应关系：

| Vue概念 | 类比   |
| ----- | ---- |
| props | 函数参数 |
| emit  | 回调函数 |

---

# 七、总结

✅ props：父 → 子（只读）
✅ emit：子 → 父（事件 + 数据）
❌ 子组件不能直接修改 props

👉 最重要一句话：

> **emit 传的数据，必须通过 `$event` 或函数参数接收，否则等于没传**

---

如果你掌握了这套机制，其实已经掌握了 Vue 组件通信的核心思想。后面再学习 `v-model`、状态管理（Pinia）都会非常轻松。
