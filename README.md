## 聊天室功能
本功能是一个基于 **B/S 架构（Browser / Server）** 的实时 Web 聊天系统，采用前后端分离开发模式，实现浏览器客户端与服务端之间的实时双向通信。

功能采用 **WebSocket 长连接通信机制** 替代传统 HTTP 请求–响应模式，使服务端能够主动向客户端推送消息，从而实现实时聊天室功能。
目前已完成以下核心功能：
1. WebSocket实时聊天  
2. 在线用户列表同步  
3. 用户上线 / 离线提醒  
4. 聊天消息广播  
5. 系统通知消息  
6. 聊天气泡UI区分（自己 / 他人）
7. 私聊功能实现啦！！！
8. 将用户系统嵌入到里面啦！
9. minio系统镶嵌里面啦！用户可以自己上传头像啦！

然后不小心写着写着就变成了我的毕设，其实非常有戏剧性。一个小小功能怎么可能变成毕设呢？！还是自己的项目管理能有点差。当时想着我以后要把所有代码都镶嵌到一个系统里面。然后会写着写着会不会变成一个很屌的系统。还是想想吧。复杂性有点太夸张了
